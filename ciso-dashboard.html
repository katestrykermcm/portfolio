Creating Meaningful Metrics

We had a lot of interesting data about the company network; the struggle was how to present it to reveal something of value. After many collaborative brainstorms, the team was able to organize the data into 3 areas: Security Governance, Vulnerability Management, and Threat Exposure. Security Governance metrics reveal how strictly a company implements standard IT practices that exist to minimize the risk of an attack, and therefore, could potentially reveal how much the company culture valued security. Vulnerability Management metrics measure how effectively IT addresses network vulnerabilities. Lastly, Threat Exposure metrics incorporate information on known malware kits to identify network vulnerabilities that are most susceptible to an attack. Together, these 3 dimensions tell a story about the network. Trending across these dimensions presents how it has changed over time.

Overview

Cyber security is a constant challenge for all modern companies. Although it is impossible to ever be completely impervious to an attack, Chief Information Security Officers (CISOs) can intelligently allocate their resources to substantially fortify their methods of protection. Like doctors, when making decisions, CISOs pair their deep expertise with relevant and current information. The CISO Dashboard, a web application for CISOs, swiftly and competently “takes the temperature” of the company's network. It equips CISOs with meaningful data helpful in assessing the “health” of their network, specifically identifying the threats and risks they face. The CISO Dashboard does not aim to be a constant monitoring tool or to provide easy answers, but rather to empower CISOs to make informed decisions.

Prototype Design

The CISO Dashboard measures the organization's “vital signs” of security risk across three dimensions: Security Governance, Vulnerability Management, and Threat Exposure. (See process work for further explanation.) The data is measured against standards set by Stroz Friedberg or the users themselves, and encapsulated in the 4 NIST tiers: Adaptive, Repeatable, Risk Informed, and Partial (in order of decreasing Cyber Resilience). The trending graph provides a clear overview of progress across each dimension, or lack thereof, over time. Upon clicking different points on the interactive graph, deeper metrics appear below and allow the user to narrow-in on specific dimensions or dates. For example, upon clicking 3 Security Governance points on the graph, the page expands and Security Governance metrics for each selected date appear. Hovering over specific metrics prompts pop-ups that explain the metrics and how they were calculated.

Delving Deeper

Navigating through the site leads to deeper analysis pages for each dimension. This interactive graph exists on a deeper analysis page for Security Governance. Changing the password standards allows the user to see how vulnerable they are to different kinds of attackers seeking user credentials through brute force.
